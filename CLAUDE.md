# CLAUDE.md

## What is this?

Kubernetes operator that auto-scales Deployments, StatefulSets, and CronJobs based on `SleepSchedule` custom resources. Saves costs by sleeping non-prod workloads on a schedule. Includes a Phoenix LiveView web UI for status and manual overrides.

Built with **Elixir + Phoenix + Bonny** (K8s operator framework). No external database — all state lives in K8s CRD conditions.

## Dev workflow

```bash
mix deps.get && mix assets.build   # first-time setup
mix phx.server                     # start locally
mix test                           # run tests
```

See `docs/MINIKUBE.md` for running against a local cluster.

## Key concepts

- **SleepSchedule CRD** — custom resource defining which workloads to sleep and when
- **Modes**: `single_namespace` (watches one NS) vs `cluster` (watches all)
- **Status conditions**: `Sleeping`, `Transitioning`, `ManualOverride`, `Error`, `Heartbeat`
- **Dependencies**: a schedule can declare `needs` (other schedules that must be awake first before this one wakes)
- Ingress is redirected to the Drowzee UI while workloads sleep (annotation-based)

## Core modules

- `Drowzee.SleepChecker` — naptime calculation (timezone-aware, cron day-of-week filter)
- `Drowzee.Controller.SleepScheduleController` — processes K8s operator events
- `Drowzee.CoordinatorSupervisor` — orchestrates scaling operations
- `Drowzee.K8s.*` — thin wrappers over K8s resources
- `DrowzeeWeb.Live.*` — Phoenix LiveView UI, updates via PubSub `sleep_schedule:updates`

## Configuration

Key env vars: `SECRET_KEY_BASE` (required in prod), `PHX_HOST`, `PORT`.
Operator config in `config/bonny.exs`. Helm values in `chart/values.yaml`.

Namespaces to manage: set via `NAMESPACES` env (`__ALL__` = cluster-wide).

## Deployment

Helm chart in `./chart/`, published to GitHub Pages.  
Raw CRD + RBAC manifest in `./manifest.yaml` (auto-generated by Bonny).  
Example SleepSchedule resources in `./k8s/`.

```bash
helm upgrade --install drowzee drowzee/drowzee --namespace default -f values.yaml
```

## CI/CD

GitHub Actions pipeline: builds Docker image → `ghcr.io/chirpwireless/drowzee:VERSION`, then packages and publishes Helm chart to gh-pages. Uses calendar-based versioning.
